# Claude-Matrix: Product Requirements Document
## Final Version - March 2026

---

## 1. Executive Summary

**Claude-Matrix** is a privacy-first, terminal-based analytics dashboard for Claude Code usage. Built with Ruby and featuring a clean TUI interface, it transforms local Claude Code data into actionable insights about development patterns, productivity, and AI usageвҖ”all without any cloud services or API calls.

### The Vision
Transform this: `~/.claude/stats-cache.json` (hidden JSON file)  
Into this: Beautiful TUI dashboard showing streaks, tool usage, work modes, and productivity patterns

### Core Principles
- **100% Local**: All data stays on your machine
- **Zero APIs**: No calls to Anthropic or any external service
- **Privacy First**: Read-only access to Claude Code data
- **Developer Friendly**: Built by developers, for developers
- **Performance**: Sub-100ms dashboard renders

---

## 2. The Opportunity

### What Exists
Claude Code stores rich usage data in `~/.claude/` including:
- Pre-computed statistics (`stats-cache.json`)
- Full session transcripts (JSONL files)
- Tool usage history
- Hourly activity patterns
- Token consumption by model

### What's Missing
- No way to visualize this data
- No insights into productivity patterns
- No understanding of tool usage
- No streak tracking or motivation
- No cost projections

### What We're Building
A beautiful terminal dashboard that answers:
- "How productive was I this week?"
- "What's my longest coding streak?"
- "Which tools do I use most?"
- "Am I in exploration or building mode?"
- "What's my busiest coding hour?"

---

## 3. Project Goals

### Primary Objectives
1. **Instant Visibility**: Launch dashboard in <100ms
2. **Useful Insights**: Surface patterns developers care about
3. **Zero Friction**: No setup, no config, just works
4. **Daily Habit**: Make checking stats addictive (streaks!)
5. **Privacy Guarantee**: Never touch network, never send data

### Success Metrics
- Parse 100% of Claude Code data accurately
- Dashboard renders in <100ms
- Tool usage matches Claude Code exactly
- Users check it daily (streak motivation)
- Zero crashes on production data

### Non-Goals (v1.0)
- вқҢ Cloud sync or backups
- вқҢ Team/multi-user features
- вқҢ Cost optimization AI
- вқҢ Integration with other tools
- вқҢ Web dashboard
- вқҢ Real-time notifications

## 5. Data Sources (In Detail)

### 5.1 Primary Source: `stats-cache.json`

**Location**: `~/.claude/stats-cache.json`  
**Updated by**: Claude Code (automatically)  
**Format**: Single JSON object

**What we get for FREE**:

```json
{
  "version": 1,
  "lastComputedDate": "2026-03-10",
  "totalSessions": 83,
  "totalMessages": 1972,
  "firstSessionDate": "2026-01-15T10:30:00.000Z",
  
  "dailyActivity": [
    {
      "date": "2026-03-10",
      "messageCount": 45,
      "sessionCount": 3,
      "toolCallCount": 89
    }
  ],
  
  "modelUsage": {
    "claude-opus-4-6": {
      "inputTokens": 134000,
      "outputTokens": 389000,
      "cacheReadInputTokens": 2150000,
      "cacheCreationInputTokens": 245000
    },
    "claude-opus-4-5-20251101": {
      "inputTokens": 117700,
      "outputTokens": 345800
    }
  },
  
  "longestSession": {
    "sessionId": "buzzing-frolicking-adleman",
    "duration": 2725776000,
    "messageCount": 757
  },
  
  "hourCounts": {
    "9": 23,
    "14": 45,
    "22": 67
  }
}
```

**Metrics we extract**:
- Sessions, messages, tokens (in/out)
- Daily activity for charts
- Model distribution
- Longest session
- Hourly activity heatmap
- First session date (for "Since" display)

### 5.2 Secondary Source: Session JSONL Files

**Location**: `~/.claude/projects/{encoded-path}/{session-uuid}.jsonl`  
**Format**: Line-delimited JSON (one event per line)

**Sample lines**:

```jsonl
{"type":"user","message":{"role":"user","content":[{"type":"text","text":"Build authentication"}]},"timestamp":"2026-03-10T14:30:00.000Z"}
{"type":"assistant","message":{"role":"assistant","content":[{"type":"text","text":"I'll help..."},{"type":"tool_use","name":"write_to_file","input":{"path":"auth.rb"}}]},"timestamp":"2026-03-10T14:30:15.000Z","usage":{"input_tokens":1234,"output_tokens":5678}}
```

**What we extract**:
- Tool usage: `write_to_file`, `bash`, `read_file`, etc.
- Session timestamps (for streak calculation)
- Message content (for prompt analysis)
- Git branches (from metadata)

**Parse once, cache forever** - these files don't change after session ends.

### 5.3 Tertiary Source: `history.jsonl`

**Location**: `~/.claude/history.jsonl`  
**Format**: One prompt per line

```jsonl
{"display":"Build user auth","timestamp":1709856234567,"project":"/Users/kapil/circle"}
{"display":"Fix the login bug","timestamp":1709856334567,"project":"/Users/kapil/circle"}
```

**What we extract**:
- Average prompt length
- Most common words ("lets", "now", "fix")
- Prompt patterns

### 6.2 Streak System

**Current Streak**: Consecutive days with at least 1 session
```ruby
def calculate_current_streak(daily_activity)
  sorted_days = daily_activity.map { |d| Date.parse(d['date']) }.sort.reverse
  
  streak = 0
  expected_date = Date.today
  
  sorted_days.each do |date|
    break if date < expected_date
    streak += 1 if date == expected_date
    expected_date -= 1
  end
  
  streak
end
```

**Longest Streak**: Maximum consecutive days in history
```ruby
def calculate_longest_streak(daily_activity)
  dates = daily_activity.map { |d| Date.parse(d['date']) }.sort
  
  max_streak = 1
  current_streak = 1
  
  dates.each_cons(2) do |date1, date2|
    if (date2 - date1).to_i == 1
      current_streak += 1
      max_streak = [max_streak, current_streak].max
    else
      current_streak = 1
    end
  end
  
  max_streak
end
```

### 6.3 Work Mode Classification

```ruby
WORK_MODES = {

def classify_work_mode(tools_used)
  total = tools_used.size.to_f
  
  WORK_MODES.transform_values do |mode|
    count = tools_used.count { |tool| mode[:tools].include?(tool) }
    ((count / total) * 100).round
  end
```

```ruby
class SessionParser
  def parse(file_path)
    tools = []
    started_at = nil
    ended_at = nil
    
    File.readlines(file_path).each do |line|
      data = JSON.parse(line)
      
      started_at ||= data['timestamp']
      ended_at = data['timestamp']
      
      if data['type'] == 'assistant'
        content = data.dig('message', 'content') || []
        content.each do |block|
          tools << block['name'] if block['type'] == 'tool_use'
        end
      end
    end
    
    {
      session_id: File.basename(file_path, '.jsonl'),
      started_at: Time.parse(started_at),
      ended_at: Time.parse(ended_at),
      tools_used: tools
    }
  end
end
```

## 10. Key Algorithms

### Streak Calculation

```ruby
def calculate_streaks(daily_activity)
  dates = daily_activity
    .map { |d| Date.parse(d['date']) }
    .sort
    .reverse
  
  current_streak = 0
  longest_streak = 0
  temp_streak = 1
  
  # Current streak
  expected = Date.today
  dates.each do |date|
    break if date < expected
    current_streak += 1 if date == expected
    expected -= 1
  end
  
  # Longest streak
  dates.sort.each_cons(2) do |d1, d2|
    if (d2 - d1).to_i == 1
      temp_streak += 1
      longest_streak = [longest_streak, temp_streak].max
    else
      temp_streak = 1
    end
  end
  
  {
    current: current_streak,
    longest: longest_streak,
    active_days: dates.size
  }
end
```

## 13. Testing Strategy

### Unit Tests

```ruby
# spec/stats_reader_spec.rb
RSpec.describe ClaudeMatrix::StatsReader do
  describe '.read' do
    it 'parses stats-cache.json correctly' do
      stats = described_class.read
      expect(stats['totalSessions']).to be_a(Integer)
      expect(stats['modelUsage']).to be_a(Hash)
    end
  end
  
  describe '.summary' do
    it 'extracts key metrics' do
      summary = described_class.summary
      expect(summary).to have_key(:sessions)
      expect(summary).to have_key(:tokens_in)
      expect(summary[:sessions]).to be > 0
    end
  end
end
```

### Integration Tests

```ruby
# spec/integration/sync_spec.rb
RSpec.describe 'claude-matrix sync' do
  it 'processes all sessions' do
    output = `bin/claude-matrix sync`
    expect(output).to include('Processed')
    expect(output).to include('sessions')
    
    # Verify database
    db = SQLite3::Database.new(cache_path)
    count = db.execute('SELECT COUNT(*) FROM sessions').first.first
    expect(count).to be > 0
  end
end
```
